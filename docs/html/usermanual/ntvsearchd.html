Running a Search Server

    • SYNOPSIS
    • DESCRIPTION
    • OPTIONS

SYNOPSIS

 ntvsearchd [-?] [-V]
            [-R resfile] [-I indexdir] [-L logfile]
            -A addr -P port
            [-u user] [-l listen-depth]

DESCRIPTION

ntvsearchd implements a nextrieve search server.

Once started, it will listen on the nominated address/port pair for incoming connections.

It will accept a stream of incoming queries in XML form from each connection, and respond with results (``hitlists'') in XML form.

A definition of what the XML for a query looks like can be found here.

A definition of what the XML for the result list looks like can be found here.

The text index to search is defined by a resource file, specified with the NTV_RESOURCE environment variable, or by using the -R option.

A definition of what the resource file contains can be found here.

OPTIONS

-R resfile
resfile specifies the location of the resource file describing the text index to be searched. This overrides any NTV_RESOURCE environment variable that is set.

-I indexdir
The indexdir specified overrides any indexdir line in the resource file. indexdir specifies the directory where the *.ntv files are located that comprise the text index. This option is useful where there are several physical text indexes using the same schema defined in a single resource file.

-L logfile
logfile explicitly overrides any logfile line in a resource file and any NTV_ERRORLOG environment variable.

-A address, -P port
Specify the machine address and port to listen on for incoming connections.

-?
Write extended help information to standard output and exit.

-V
Write version information to standard output and exit.
